# Competitor Research — Turkish Driving School Management Software

Date: 2026-05-06

## Our Product (mtsk.app)

Cloud + desktop platform for Turkish MTSK driving schools:
- MEBBIS integration (browser automation against mebbis.meb.gov.tr)
- Student & vehicle management, multi-role auth (Super Admin, Admin, Owner, Manager)
- PDF generation (Direksiyon Takip, Simulasyon Raporu) via PHP/RabbitMQ worker
- Electron desktop app — multi-account MEBBIS sessions, auto-login, batch PDF downloads
- Modern React + TypeScript SPA, NestJS microservices
- Real-time WebSocket notifications

## MEBBIS Integration Verdicts

Legend: **A = Scrapes MEBBIS** | **B = No MEBBIS integration** | **C = Manual file import/export** | **D = Unknown**

**Critical industry finding: NO official MEB/MEBBIS API exists.** Every vendor that integrates does so via browser automation / web scraping. Confirmed by Tabim's public changelog literally documenting rewriting their integration after MEB changed activation/security codes (2FA).

## Strategic Findings

1. **Only the 4 legacy desktop products (AKINSOFT, WENNTEC, Makrokey, Tabim) integrate with MEBBIS — all 4 by scraping**, not API. Every MEB UI/2FA change breaks them. This is the competitive moat we share with them, but they're stuck on Windows desktop.

2. **All 5 "modern/web" competitors (kursum, surucuegitim, drkdersi, e-surucu, drivbi) avoid MEBBIS entirely.** They're student-facing portals, exam-prep content, or website builders. None solve the actual MEBBIS pain.

3. **Our differentiation remains unique:** Modern web SaaS + deep MEBBIS automation + Electron desktop batch tool. No competitor combines all three.

4. **Pricing is opaque industry-wide.** Only 2 of 9 publish numbers (E-Sürücü per-pack, Drivbi 3,450 TL one-time). The 4 MEBBIS-scrapers all use dealer/call-for-quote. Real benchmarking requires sales-call data gathering.
