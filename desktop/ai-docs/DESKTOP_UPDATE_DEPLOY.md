# Desktop App Update Deployment Guide

## CRITICAL: Mandatory Update Policy
**Users can NEVER use the app unless they are on the latest version.** This is the fundamental rule of the update system. Every deployment MUST update `minimum_version.json` to the new version number so that all older versions are blocked immediately. There is no concept of "optional" updates — all updates are mandatory.

## Overview
The desktop app (Electron) uses a mandatory update system:
1. **Version Gate** — blocks the app if below minimum version (which is ALWAYS set to the latest version)
2. **Auto-Updater** — `electron-updater` handles download + install of new versions

All update files are hosted at `https://online.mtsk.app/desktop-updates/` on the server `mtsk@ekullanici_yeni`.

## Server File Structure
```
/home/mtsk/online.mtsk.app/desktop-updates/
├── minimum_version.json          ← Version gate config
├── latest.yml                    ← electron-updater metadata (auto-generated by electron-builder)
├── MTSK_APP Setup X.X.X.exe      ← Windows installer
└── MTSK_APP Setup X.X.X.exe.blockmap  ← Delta update support
```

### minimum_version.json
```json
{
  "minimumVersion": "1.0.3",
  "message": "Bu surum artik desteklenmiyor. Uygulamayi kullanabilmek icin guncelleme yapmaniz gerekmektedir."
}
```
- `minimumVersion`: App versions below this are **blocked** on startup
- `message`: Shown in the mandatory update dialog (optional, has default)

### latest.yml
Auto-generated by `electron-builder`. Contains version, file URL, sha512 hash, size, and release date. **Do not edit manually.**

## How Updates Work

### On App Startup
1. App fetches `minimum_version.json` from `https://online.mtsk.app/desktop-updates/`
2. Compares local version (from `package.json`) against `minimumVersion`
3. If **below minimum** → shows blocking dialog → calls `checkForUpdates()` → downloads exe → installs → restarts
4. If **allowed** → sets up optional auto-updater that checks `latest.yml` and prompts user
5. If **server unreachable** → allows app to run (fail-open)

### Update Policy (ALWAYS MANDATORY)
All updates are mandatory. Every deployment MUST set `minimumVersion` in `minimum_version.json` to the new version. Users cannot use the app unless they are on the latest version. There is no optional update flow.

## Step-by-Step Deploy

### 1. Bump version in package.json
```
cd desktop
# Edit package.json "version" field, e.g. "1.0.4"
```

### 2. Build the installer
```bash
cd desktop
npm run dist
```
Output goes to `desktop/release/`:
- `MTSK_APP Setup X.X.X.exe`
- `MTSK_APP Setup X.X.X.exe.blockmap`
- `latest.yml`

### 3. Clean old files on server
```bash
ssh mtsk@ekullanici_yeni "rm -f /home/mtsk/online.mtsk.app/desktop-updates/'MTSK_APP Setup'* /home/mtsk/online.mtsk.app/desktop-updates/latest.yml"
```

### 4. Upload new files
```bash
scp "desktop/release/latest.yml" "desktop/release/MTSK_APP Setup X.X.X.exe.blockmap" mtsk@ekullanici_yeni:/home/mtsk/online.mtsk.app/desktop-updates/

scp "desktop/release/MTSK_APP Setup X.X.X.exe" mtsk@ekullanici_yeni:/home/mtsk/online.mtsk.app/desktop-updates/
```

### 5. Update minimum_version.json (REQUIRED)
Edit `desktop/release/minimum_version.json` to match the new version:
```json
{
  "minimumVersion": "X.X.X",
  "message": "Bu surum artik desteklenmiyor. Uygulamayi kullanabilmek icin guncelleme yapmaniz gerekmektedir."
}
```
Then upload:
```bash
scp "desktop/release/minimum_version.json" mtsk@ekullanici_yeni:/home/mtsk/online.mtsk.app/desktop-updates/
```
**This step is NOT optional.** Every deployment must update `minimumVersion` to the new version.

### 6. Verify
```bash
ssh mtsk@ekullanici_yeni "ls -lh /home/mtsk/online.mtsk.app/desktop-updates/"
```

## Trick Points
- **Filename has spaces**: `MTSK_APP Setup X.X.X.exe` — use quotes in SCP commands
- **checkForUpdates() before downloadUpdate()**: `electron-updater` requires checking first, then downloading on `update-available` event. Calling `downloadUpdate()` directly fails with "Please check update first"
- **Server unreachable = fail-open**: If the version check URL is down, the app runs normally (can't gate without server)
- **Old files must be removed**: Keep only the current version's exe/blockmap on the server to avoid confusion
